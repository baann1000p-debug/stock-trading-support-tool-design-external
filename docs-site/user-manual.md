# 操作手順書 (User Manual)

> **ドキュメントID**: MANUAL-EXT-001
> **対象システム**: 個人投資家向け投資判断支援ツール (Investment Decision Support Tool)
> **最終更新日**: 2026-07-23

---

## 1. はじめに

本書は、**個人投資家向け投資判断支援ツール**の操作手順書です。
ログインからダッシュボードでの情報収集、適切な購入株数を割り出すポジションサイジング計算、エントリー中銘柄のリスク管理（トレーリングストップ・出口シグナル）、取引理由の記録（投資日記）、および自動分析バッチの運用手順までを分かりやすく解説します。

---

## 2. 初期セットアップ & ログイン手順

### 2.1 初期アカウントの作成 (初回のみ)

システムのダッシュボードにログインするためのユーザーを作成します。ターミナル（コマンドプロンプトやPowerShell）を開き、以下のコマンドを実行します。

```bash
# ユーザー追加コマンド実行例: ユーザー名 "investor1", パスワード "MyPassword123"
node src/workflow/add-user.js investor1 MyPassword123
```

成功すると、`ユーザー 'investor1' を作成しました。` と表示されます。

### 2.2 サーバーの起動

開発モードでバックエンド API とフロントエンド開発サーバーを起動します。

```bash
# ターミナル1: バックエンド API サーバー起動 (Port: 3000)
npm start

# ターミナル2: フロントエンド Vite 開発サーバー起動 (Port: 5173)
npm run dev
```

### 2.3 画面へのアクセスとログイン

1. Webブラウザ（Google Chrome / Microsoft Edge 等）を起動し、`http://localhost:5173` へアクセスします。
2. ログイン画面が表示されます。ユーザー名とパスワードを入力するフォームが表示されます。
3. 2.1 で作成した **ユーザー名** と **パスワード** を入力し、「ログイン」ボタンをクリックします。
4. 認証が成功すると、ダッシュボード画面へ移動します。

## 3. Web UI 各画面の操作ガイド

### 3.1 共通ナビゲーション

画面左側のサイドバーメニューから、目的の機能画面を選択できます。

| メニュー項目 | 説明 |
|---|---|
| 📊 **ダッシュボード** | マクロ景気・AIアロケーション・市場動向 |
| 🎯 **推奨銘柄** | スクリーニング厳選銘柄一覧 |
| 💼 **エントリー中銘柄** | 保有ポジション・リスク管理・出口シグナル |
| 📈 **チャート分析** | テクニカルチャート |
| 📝 **運用状況** | 取引履歴・投資日記・CSVダウンロード |
| 📄 **過去のAIレポート** | アーカイブ閲覧 |

---

### 3.2 ダッシュボードの閲覧

ログイン後の初期画面です。

1. **最新のマクロ経済環境**:
   - AIが判定した現在の「景気フェーズ」（例：回復期 / 拡大期）および「推奨アセットアロケーション比率」を確認します。
2. **ソーシャルトレンド分析**:
   - 投資家の関心ワードやポジティブ/ネガティブ比率の解説を確認し、市場のセンチメントを把握します。

::: tip 活用ポイント
景気フェーズが「後退期」の場合は株式比率を下げ、「回復期」に入ったタイミングで買いポジションを増やすと効果的です。
:::

---

### 3.3 推奨銘柄の確認 & 検討

スクリーニングバッチによって抽出された注目銘柄を確認します。

1. サイドバーから **推奨銘柄** を選択します。
2. 表示された銘柄一覧テーブルで、PER・PBR・ROE・配当利回り・AI評価スコアを確認します。
3. 買付を検討したい銘柄がある場合は、操作列の **「ポジションサイジング」** ボタンを押すか、**「保有銘柄に追加」** ボタンをクリックします。

---

### 3.4 ポジションサイジング計算機による購入株数算出

適切な資金管理を行い、1回の取引での大損を防ぐために必ず使用します。

**手順:**

1. 「推奨銘柄」または「エントリー中銘柄」画面で「ポジションサイズ計算」ボタンをクリック。
2. ポジションサイジングダイアログが開きます。
3. 以下の4項目を入力します：

| 入力項目 | 入力例 | 説明 |
|---|---|---|
| 総投資資産額 | 5,000,000 円 | 現在の運用資産の合計 |
| 許容リスク率 | 1.0 % | 1回の取引で失ってよい資産の割合 |
| エントリー予定株価 | 2,500 円 | 買い付け予定の株価 |
| 損切り設定株価 | 2,300 円 | 損切りを行う株価の下限 |

4. 「計算実行」ボタンをクリック。
5. 出力結果を確認します：

| 出力項目 | 計算例 |
|---|---|
| 許容最大損失額 | 50,000 円 |
| 推奨購入株数 | 200 株 (100株単位調整済) |
| 必要投資金額 | 500,000 円 |
| ポートフォリオ比率 | 10% |

::: warning 注意
損切りラインは必ず設定してください。リスク率は通常 **1〜2%** に設定することを推奨します。
:::

### 3.5 エントリー中銘柄の管理 & 出口シグナル対応

保有中ポジションのリスク管理および出口（利益確定・損切り）の判断を行います。

1. サイドバーから **エントリー中銘柄** を選択します。
2. **トレーリングストップの確認・調整**:
   - 株価が上昇した場合、最高値に合わせて「損切ライン（ストップ値）」を自動切り上げ更新します。
3. **出口シグナルの確認**:
   - 以下の警告バッジが表示されている場合、記載の対応を実施してください。

| 警告バッジ | 発生条件 | 推奨アクション |
| ----------------------------- | ------------------------- | -------------------------------------------------------------------- |
| 🔴 **目標株価到達** | 現在値 ≧ 設定した目標株価 | 利益確定（全売却または半値利確）を検討します。 |
| 🔴 **損切りライン到達** | 現在値 ≤ 損切ライン | 躊躇せず機械的に損切り注文（成行/指値）を発注します。 |
| 🟡 **RSI買われすぎ (70以上)** | 14日RSI ≧ 70 | 買われすぎによる一時調整に備え、トレーリングストップを引き上げます。 |
| 🟡 **決算発表直前** | 決算日まで 3 日以内 | 決算跨ぎのリスクを考慮し、ポジション縮小または撤退を検討します。 |

---

### 3.6 テクニカルチャート分析

銘柄の株価推移やテクニカル指標をビジュアルで分析します。

1. サイドバーから **チャート分析** を選択します。
2. 画面上部の銘柄検索窓に銘柄コード（例: `7203`）を入力します。
3. 表示切り替えスイッチで移動平均線（5日/25日/75日）、ボリンジャーバンド、RSIを表示/非表示にします。

---

### 3.7 運用状況の登録 & 投資日記の記録

取引結果と投資判断の振り返りを記録し、トレードスキルの向上に役立てます。

#### 取引の新規登録

1. サイドバーから **運用状況** を選択します。
2. 画面右上の **「新規取引追加」** ボタンをクリックします。
3. 日付、銘柄コード、売買種別（買/売）、株数、単価を入力して保存します。

#### 投資日記（振り返りメモ）の記録

1. 登録された取引行の **「日記編集」** アイコンをクリックします。
2. 以下の項目を入力・更新して保存します：
   - **エントリー理由**: なぜこのタイミングで買ったか（ファンダメンタルズ/テクニカル/AI推してなど）
   - **エグジット理由**: なぜ売却したか（目標到達/損切り/決算回避など）
   - **振り返り・反省点**: ルール通り運用できたか、次回の改善点
3. 全データをバックアップ・分析したい場合は **「CSVエクスポート」** ボタンでファイルをダウンロードできます。

---

### 3.8 過去のAIレポートの閲覧

これまでに自動生成された日次AIレポートを参照します。

1. サイドバーから **過去のAIレポート** を選択します。
2. 日付一覧から参照したいレポートを選択すると、当時の景気判断・AI注目セクター・推奨銘柄のバックナンバーを閲覧できます。

## 4. ワークフローバッチの実行・運用手順

市場データの自動取得、AI分析、およびHTMLメール通知を行うバッチの運用方法です。

### 4.1 手動テスト実行 (メール送信なし)

動作確認を行う場合は、メール送信を行わないテストモードで実行します。

```bash
# マクロ分析および推奨銘柄抽出のテスト実行
npm run test-workflow -- --mode full

# 推奨銘柄抽出のみのテスト実行
npm run test-workflow -- --mode recommend
```

### 4.2 本番実行 (データベース保存 & メール送信)

市場前・市場後等に手動で本番バッチを動かすコマンドです。

```bash
# 日次マクロ分析バッチ実行
npm run workflow:macro

# 推奨銘柄スクリーニング & AI分析バッチ実行
npm run workflow:recommend
```

### 4.3 定期自動化 (Cron 設定例)

Linux/Mac の Cron や Windows のタスクスケジューラを使用して、平日朝（8:30）や夕方（17:30）に自動実行させる設定例です。

```txt
# 例: 平日(月〜金) 17:30 に日次推奨銘柄バッチを実行
30 17 * * 1-5 cd /path/to/stock-trading-support-tool && /usr/bin/npm run workflow:recommend >> /path/to/logs/cron.log 2>&1
```

## 5. FAQ・トラブルシューティング

### Q1. ログイン時に「認証に失敗しました」と表示される

- ユーザー名およびパスワードが正しく入力されているか確認してください。
- ユーザー未作成の場合は `node src/workflow/add-user.js <username> <password>` で作成してください。

### Q2. 推奨銘柄やマクロ指標が表示されない / エラーになる

- `.env` ファイル内の `GEMINI_API_KEY` 等の外部APIキーが正しいか確認してください。
- ネットワーク接続およびYahoo!ファイナンス等の外部サービスが正常に稼働しているか確認してください。

### Q3. メールが届かない

- `.env` の SMTP 設定（`SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`）を確認してください。
- メールソフトの迷惑メールフォルダに分類されていないか確認してください。
